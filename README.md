    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-525icons-v140" class="anchor" href="#525icons-v140" aria-hidden="true"><span class="octicon octicon-link"></span></a><a href="http://525icons.com">525icons v1.4.0</a></h1>

<h3><a id="user-content-a-web-icon-font-with-style" class="anchor" href="#a-web-icon-font-with-style" aria-hidden="true"><span class="octicon octicon-link"></span></a>A web icon font with style.</h3>

<p>525icons is a very comprehensive and versatile web icon collection, designed for websites and apps, and it is completely free for commercial use. </p>

<p>Most of the icons are designed, customized and vectorized "by hand" in Adobe Illustrator before the processing into a font.
A few are directly copied from the different open source icon packs that are available online, 
among them the "IcoMoon - Free" pack and the "Font Awesome" icon collection. Some are known brand logos, 
vectorized from png format or copied from open source svg files found on the net. </p>

<p>The font is assembled in the icoMoon app: <a href="https://icomoon.io/app">https://icomoon.io/app</a>.</p>

<h2><a id="user-content-instructions-how-to" class="anchor" href="#instructions-how-to" aria-hidden="true"><span class="octicon octicon-link"></span></a>Instructions (How-to)</h2>

<p>If you are familiar with the use of the Font Awesome web font, the 525icons web font css code has some differences.</p>

<p>The icons are placed by adding the 'ico-' class prefix to the icons name (Font Awesome uses the 'fa-' prefix).
Transitions, sizes and effects use the 'ux-' prefix (While the Font Awesome uses the 'fa-' prefix all over). </p>

<p>Make sure the entire fonts directory is copied into your project.
You will need the files:</p>

<ul>
<li>fonts/525icons.css</li>
<li>fonts/525icons.eot</li>
<li>fonts/525icons.svg</li>
<li>fonts/525icons.ttf</li>
<li>fonts/525icons.woff</li>
<li>fonts/525icons.woff2</li>
</ul>

<p>You do not need the file 525icons.otf, unless you want to install it on your OS.
The css file must be in the same directory as the fonts. If not, you have the make changes in the code.</p>

<p>In the <code>&lt;head&gt;</code> of your html, reference the location to your 525icons.css and fonts. </p>

<p><code>&lt;link rel="stylesheet" type='text/css' href="path/to/fonts/525icons.css"&gt;</code></p>

<p><strong>How to place the icons in your html-file:</strong><br>
Like Font Awesome, 525icons is designed to be used with the inline element: <code>&lt;i&gt;</code> 
You can place the icons using the CSS Prefix 'ico-' and the icon's name, like this:
<code>&lt;i class="ico-yin-yang"&gt;&lt;/i&gt;</code> </p>

<p>And you may add additional classes, like this example:</p>

<p><code>&lt;i class="ico-yin-yang ux-lg" style="color: #000;"&gt;&lt;/i&gt;</code></p>

<p>You may further investigate all the different css-codes in use, inside the file: <a href="/runestro/525icons/blob/master/demo.html">demo.html
</a> and  <a href="/runestro/525icons/blob/master/fonts/525icons.css">525icons.css</a>.</p>

<p>To use the 525icons OTF font in desktop applications, you have to install the font on your system. 
When installing the font or viewing it, the operating system may not show all the glyphs in the font. In most cases only the basic Latin characters are shown.</p>

<p>The 525icons icon set is optimized for (16 × N)px sizes. You will get crisp results by setting your font-size to 16px, 32px, 48px (= 3 × 16px), etc.</p>

<p>The .ux-Nx classes are optimized for these sizes, like this:</p>

<p><code>.ux-2x {font-size: 32px;}</code></p>
